# Система мониторинга газотурбинной установки (ГТУ) – КМ №4

Серверное приложение для мониторинга параметров газотурбинной установки с имитацией датчиков, автоматическим определением режимов работы, детекцией аномалий, аутентификацией пользователей и полным аудитом действий.

## Основные возможности

- Имитация работы датчиков ГТУ (обороты, температура выхлопных газов, давление на входе, расход топлива, вибрация, положение IGA) в реальном времени.
- Автоматическая смена режимов работы ГТУ (STOP → START → IDLE → PARTIAL → NOMINAL → EMERGENCY → STOP) по заданному циклу.
- Классификация текущего режима по оборотам и сравнение всех параметров с нормативными диапазонами.
- Выявление аномалий – параметров, выходящих за пределы допусков.
- Сохранение всех показаний и аномалий в SQLite-базу данных.
- REST API для получения текущего состояния и истории.
- Аутентификация пользователей с помощью JWT-токенов.
- Разграничение прав: обычный пользователь и администратор.
- Блокировка аккаунта после неудачных попыток входа, ограничение частоты запросов (rate limiting).
- Полный аудит событий: успешные и неудачные входы, смена паролей, доступ к данным, аномалии, смена режимов, создание пользователей.
- Ведение журналов (логов) с ротацией файлов и цветным выводом в консоль.
- Проверка целостности записей аудита с помощью контрольных сумм (SHA-256).

## Технологический стек

- **Язык:** Python 3.14+
- **Сервер:** FastAPI + Uvicorn
- **Базы данных:** SQLite (три отдельные БД: для датчиков, для пользователей, для аудита)
- **Аутентификация:** PyJWT + bcrypt
- **Логирование:** встроенный модуль `logging` с ротацией
- **Тестирование:** unittest
- **Интерфейс:** PyQt6

## 1. Установка зависимостей
```bash
pip install fastapi uvicorn bcrypt pyjwt pyqt6
```

## 2. Запуск сервера
```bash
python server.py
```

Сервер автоматически выберет свободный порт (начиная с 8000). По умолчанию создается учетная запись администратора:
**Логин**: ```admin```
**Пароль**: ```Admin@12345```

Также создается три SQLite-файла
  * ```gtu_data.db``` - Показания датчиков и аномалии.
  * ```gtu_auth.db``` - База пользователей.
  * ```audit.db``` - События аудита.

Чтобы проверить работоспособность сервера:
```http://localhost:8000/health``` в браузере. Должно вернуться ```{"status": "ok"}```.

## 3. Запуск клиента
```bash
python main_ui_result.py
```

Открывается окно аутентификации. После ввода логина и пароля пользователь получает доступ к мониторингу
